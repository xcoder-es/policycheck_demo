Technical Overview

Lightweight architecture for a practical POC

Architecture approach

Built as a lightweight modular monolith with clear domain boundaries and ports/adapters.

AI usage

AI is isolated behind adapters and has deterministic fallback paths.

Validation approach

Compliance checks are deterministic and rule-based once controls are reviewed.

Deployment model

The demo avoids local model loading, databases, background workers and heavy processing.

Data handling

This POC uses transient demo data and does not persist uploaded documents or bordereaux records.

Limitations

This is a POC, not a production compliance engine. Production would require security, persistence, governance and integration work.
